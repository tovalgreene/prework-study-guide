Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Stands for Hypertext Markup Language, one of the three cornerstone languages used in web development, used to create the structure of a webpage
  • Elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage
  • Elements used to add content to pages: text, image, header, list
  • Elements used to divide and describe a page: meta-related, content division, semantic
  • Head element contains metadata that helps the browser render the page correctly; not visible when a user looks at a webpage
  • Body element contains the content shown to the user; all the elements that we want the user to see and interact with must be placed inside this element
  • Metadata tags: title defines title for webpage, style defines CSS styles associated w/ website, base defines base URL, link connects an external resource to the HTML document, meta defines metadata such as character set, description, keywords, author, viewpoint, script embeds client-side scripts in the HTML document
  • Charset sets character encoding, http-equiv sets an HTTP header for content, viewport sets default view
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag

CSS

  • Stands for Cascading Style Sheets, gives the content style

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Github is a cloud-based service that allows developers to store, manage, and collaborate on code
  • Git is the software on your computer that tracks changes that you make to your code
  • A commit is a change to your code with an identity -- change the name that's associated with your Git commits using the "git config" command
  • Version control - "git revert"

  • "README" files are used to pass along useful information to anyone who visits the repo on GitHub

General Notes

  • ls is list, cd is change directories, pwd is print working directory, SSH key is an access credential for the secure shell network protocol, mkdir is make directory
  • Expected Behavior: User Story and Acceptance Criteria
  • A feature is a functional part of the application or website that has value or fulfills a need in the business
  • A GUI or graphical user interface is a way for users to interact with the computer visually
  • A branch is a copy or version of the code that can be worked on independently in isolation so development changes to not disrupt other team members who are also working on the code and testing their solutions
  • Default branch is called main
  • A pull request or PR is used to merge a comparison branch into the base branch
