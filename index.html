HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Stands for Hypertext Markup Language, one of the three cornerstone languages used in web development, used to create the structure of a webpage
  • Elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage
  • Elements used to add content to pages: text, image, header, list
  • Elements used to divide and describe a page: meta-related, content division, semantic
  • Head element contains metadata that helps the browser render the page correctly; not visible when a user looks at a webpage
  • Body element contains the content shown to the user; all the elements that we want the user to see and interact with must be placed inside this element
  • Metadata tags: title defines title for webpage, style defines CSS styles associated w/ website, base defines base URL, link connects an external resource to the HTML document, meta defines metadata such as character set, description, keywords, author, viewpoint, script embeds client-side scripts in the HTML document
  • Charset sets character encoding, http-equiv sets an HTTP header for content, viewport sets default view
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag

CSS

  • Stands for Cascading Style Sheets, gives the content style
  • Three ways to style a webpage using CSS: inline, internal style sheet, external style sheet
  • Inline: developer wants to style an element directly in the HTML file
  • Internal style sheet - embed style sheet directly in HTML file
  • External style sheet - allows developers to keep CSS rules in a separate file, which makes design changes easier
  • Declarations contain two components: property we want to apply and value of the property
  • Margin indicates how much space we want around the outside of an element - adds a buffer
  • Padding adds space around the content inside an element
  • link element creates connection to external file
  • Rel specifies relationship between current document and linked document resource
  • href specifies the URL of the external resource, using a relative path
  • HTML color codes - https://htmlcolorcodes.com/
  • Period after card designates class selector - any HTML element assigned to the class card will gain these property assignments from the CSS
  • Resources - https://developer.mozilla.org/en-US/docs/Web/CSS

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Github is a cloud-based service that allows developers to store, manage, and collaborate on code
  • Git is the software on your computer that tracks changes that you make to your code
  • A commit is a change to your code with an identity -- change the name that's associated with your Git commits using the "git config" command
  • Version control - "git revert"

JavaScript

  • Allows users to interact with a website
  • Control flow - the order in which a computer executes code in a JavaScript file
  • Varluables are containers that store values - start by declaring a variable with the let keyword
  • A semicolon at the end of the line indicates where a statement ends
  • JavaScript is case sensitive
  • String - sequence of text enclosed in quotation marks
  • Numbers don't have quotes around them
  • True and false--Boolean--are keywords that don't need quote marks
  • Array - structure that allows you to store multiple values in a single reference
  • Everything in JavaScript is an object and can be stored in a variable
  • Comments are snippets of text that can be added along with code
  • An operator is a mathematical symbol that produces a result based on two values or variables; + combines two numbers or strings, = assigns a value to a variable
  • JavaScript expressions and operators: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators
  • Conditionals are code structures used to test if an expression returns true or not - if...else is an example
  • Functions are a way of packaging functionality that you wish to reuse
  • Events are code structures that listen for activity in the browser and run code in response
  • JavaScript language overview - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Language_overview
  • Dynamic language assocated with dynamic types
  • Weakly typed language - allows implicit type conversion when an operation involves mismatched types
  • JavaScript data types and data structures - https://developer.mozilla.org/en-US/docs/Web/JavaScript/Data_structures
  • A variable is a named container that allows us to store data in our code
  • A string is a variable that includes all of the topics, within quotation marks
  • Console refers to a test environemnt that developers use to check out their code
  • Log is a method, a set of instructions that can be executed by a computer
  • Control flow is the order in which a computer executes code in a script
  • Use conditional statements and loops to control the order in which the code is executed, or if it is even executed at all
  • An if statement is a way of introducing decision-making into our code
  • Truthy vs falsy - https://developer.mozilla.org/en-US/docs/Glossary/Truthy
  • An array is a single variable that is used to hold a group of data
  • Each piece of data's location number is called their index
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
  • For loop contains three statements - starting point, condition, final statement that allows the array to iterate over each item

General Notes

  • ls is list, cd is change directories, pwd is print working directory, SSH key is an access credential for the secure shell network protocol, mkdir is make directory
  • Expected Behavior: User Story and Acceptance Criteria
  • A feature is a functional part of the application or website that has value or fulfills a need in the business
  • A GUI or graphical user interface is a way for users to interact with the computer visually
  • A branch is a copy or version of the code that can be worked on independently in isolation so development changes to not disrupt other team members who are also working on the code and testing their solutions
  • Default branch is called main
  • A pull request or PR is used to merge a comparison branch into the base branch
  • Separation of concerns refers to separating code into different files based on use
  • Do Not Repeat Yourself or DRY is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained
